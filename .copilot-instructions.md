# Copilot Instructions (Windwalker 生態多專案通用)

## 目標
- 優先產出可維護、可測試、向後相容的 PHP 程式碼。
- 變更以最小必要範圍為原則，不做與需求無關的重構。

## 技術基準
- 語言版本以 **PHP 8.4+** 能力為準（使用嚴格型別與現代語法）。
- 套件管理使用 **Composer**。
- 介面與相容性盡量遵循 PSR（Container、HTTP Message、Cache、Event Dispatcher 等）。

## 專案結構慣例
- 採用 `packages/<package>/src` 與 `packages/<package>/test` 的對應結構。
- 新增功能時，優先放在既有 package 的邊界內；跨 package 依賴要明確且最小化。
- 命名空間與目錄保持一致（PSR-4）。

## 程式碼風格
- 所有 PHP 檔案使用 `declare(strict_types=1);`。
- 風格遵循 **PSR-12**，保持與既有檔案一致（縮排、空行、import 排序、尾逗號慣例）。
- 優先使用明確型別（參數、回傳、屬性）；避免不必要的 `mixed`。
- 例外處理使用具語意的例外類別，不吞錯誤。
- 避免引入全域副作用；優先純函式與可注入依賴。

## 設計原則
- 優先維持公開 API 相容性；若必須破壞相容，需明確標示影響範圍。
- 既有不可變模式（例如 `with*()`）需維持不可變語意，不在原物件上直接修改狀態。
- 優先組合（composition）而非繼承（inheritance）。
- 抽象層只在有明確重用價值時新增，避免過度設計。

## 測試策略
- 新增/修改行為需同步補上或更新 PHPUnit 測試，放在對應 package 的 `test` 目錄。
- 測試命名需描述行為而非實作細節，並覆蓋正常路徑與邊界條件。
- 若功能涉及 DB/HTTP/快取等整合元件，盡量隔離外部依賴並使用既有測試基礎設施。

## 驗證指令（依專案實際環境）
- 安裝依賴：`composer install`（無 lock 時可用 `composer update`）。
- 程式碼風格：`vendor/bin/phpcs packages -n --extensions=php --standard=PSR12`
- 測試：`php vendor/bin/phpunit --configuration phpunit.ci.xml`

## 變更守則
- 不提交秘密資訊、金鑰或憑證。
- 不任意新增重量級依賴；能用現有元件就不要重複造輪子。
- 文件、型別、測試需與程式碼同步更新。
- 提交前確認只包含本需求相關檔案與差異。
